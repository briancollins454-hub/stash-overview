Understanding Status Badges

Orders display two key status badges — one from Shopify (fulfilment) and one from DecoNetwork (production). Understanding these is essential.

Shopify Fulfilment Status

This shows whether the order has been shipped to the customer:

BadgeMeaningWhat To Do
Fulfilled All items have been dispatched and tracking sent to customer. Nothing — this order is complete.
Partial Some items shipped but others remain. Common with split shipments. Check the expanded view to see which items are still outstanding.
Unfulfilled Nothing has been shipped yet. Standard state — this order needs production and/or shipping.

Days Remaining Progress Bar

The coloured bar next to the dispatch date tells you how much time is left:

ColourMeaning
GreenOn track — More than 10 working days remaining.
BlueApproaching — 6–10 working days left. Keep an eye on it.
OrangeUrgent — 5 or fewer days. Needs priority attention.
RedOverdue — Past the dispatch date. Immediate action required.

Production Status Meanings

The Production Status badge shows where the order is in the DecoNetwork production pipeline. This is the most important indicator for daily workflow decisions.

StatusBadgeWhat It Means
Not Ordered Not Ordered The Shopify order exists but no production order has been placed with Deco. The customer has ordered, but work hasn't started. This needs a Deco job to be created.
Awaiting Deco Data... Awaiting Deco Data... The system is actively syncing with DecoNetwork to pull in production data. This is a temporary state — it will update automatically once the sync completes. No action needed, just wait.
Awaiting Deco Detail... Awaiting Deco Detail... Similar to above — the system has identified a Deco job but is fetching the detailed item-level data. Wait for sync to finish.
Awaiting Stock Awaiting Stock Deco has the order but is waiting for blank garments/stock to arrive before production can begin.
Awaiting Processing Awaiting Processing Stock is in but the job is queued for production. It's in the pipeline but not yet on the machines.
In Production In Production Items are actively being decorated — embroidery, printing, or other processes underway.
Quality / Process Quality Production complete, items are in quality checking or a specialist process stage.
Artwork / Quote / PO Artwork Pre-production phase — artwork needs approval, quote needs confirmation, or purchase order needs raising.
Ready / Completed Ready Production finished — items are ready to be dispatched. Head to the FULFILL tab to ship.
Shipped Shipped Deco has dispatched the items. Check ShipStation for tracking details.
Identified / Linked Linked The Shopify order has been matched to a Deco job but detailed production status hasn't been pulled yet.
Cancelled / On Hold Cancelled Order has been cancelled or put on hold at the Deco level.
Unknown Unknown The system couldn't determine the production status. May need a manual check or re-sync.
⚠️ Common Confusion
"Awaiting Deco Data..." does NOT mean the order hasn't been placed. It means the app is still loading data from DecoNetwork. If it persists for more than a few minutes, try hitting STATUS REFRESH or SYNC.

Dashboard — The Order Table

Dashboard Tab

The Dashboard is your central command post. Every Shopify order appears here, grouped by club or vendor, with all linked Deco data alongside it.

Table Columns

ColumnWhat It Shows
☑Selection checkbox — tick to include in batch actions.
Shopify Order #Order number, customer name, date, and club tags. Click the order number to open in Shopify admin.
Deco Job #Linked DecoNetwork job ID. Shows Find Job if not yet linked.

Searching & Filtering

  • Search bar — Type an order number, customer name, job ID, or tag. Search is instant (debounced).
  • Filter Tags — Click the dropdown to filter by specific clubs or teams.
  • Date Range — Set start/end dates to narrow the view to a specific period.
  • Show Fulfilled — Toggle this on to include completed orders in the table.
  • Group by Vendor — Switch from club grouping to vendor grouping.

Grouping

Orders are automatically grouped by club/team name (from Shopify tags). Each group shows a header with the club name and a green badge showing the number of instances. You can toggle to vendor grouping to see orders by supplier instead.

ℹ️ Saved Views
Click + Save View to save your current combination of filters, search term, and date range. Name it (e.g., "Overdue AFC") and click it anytime to instantly restore those filters.

Summary Cards & Quick Filters

Dashboard Tab

The five summary cards at the top of the Dashboard give you an instant overview and act as one-click filters:

🔴 Not On Deco

Orders that have no linked Deco job at all. These customers have ordered on Shopify but no production has been set up. Sub-filters: 5+ Days (orders placed more than 5 days ago without a job) and 10+ Days (even more overdue).

🟠 Mapping Gaps

Orders that have a Deco job linked but the individual line items haven't been mapped to specific Deco products yet. The warning text "Job ID exists but items need mapping to track progress" means you need to open the order and run item mapping.

🟢 Ready to Ship

Production is 100% complete — these orders are sitting on the shelf waiting to be packed and shipped. Sub-filters: Order Complete (completely done), Stock Ready (stock items available), Partially Ready (some items ready, slider to set threshold).

🔵 Unfulfilled

All orders that haven't been shipped yet. Sub-filters: Late (20D+) (unfulfilled for 20+ days — investigate immediately), Production After Dispatch (Deco finished but not yet shipped — why?), Due Soon (approaching dispatch deadline).

🟢 Fulfilled (7D)

Orders shipped in the last 7 days. Useful for reviewing recent dispatches. Sub-filter: Recent Activity and count of Partial shipments that still need completing.

💡 How To Use
Click any summary card to instantly filter the table below to just those orders. Click the sub-filter checkboxes for more specific views. Click the card again (or clear filters) to return to the full list.

Understanding Deco Job Linking

The core purpose of Stash Shop Overview is to bridge Shopify orders with DecoNetwork production jobs. Here's how the linking lifecycle works:

The Journey of an Order

1
Customer places order on Shopify — appears in the Dashboard as Not Ordered with no Deco Job #.
2
Production job created in DecoNetwork — a team member creates the job manually in Deco.
3
Job linked in the app — either via auto-scan, manual entry, or the LINKER tab. Status changes to Linked.
4
Items mapped — individual Shopify line items matched to specific Deco products. Completion % starts tracking.
5
Production progresses — status updates automatically from Deco: Awaiting Stock → In Production → Ready.
6
Order shipped — fulfilled via ShipStation in the FULFILL tab or directly in Shopify.
🎯 MTO (Make-To-Order) Items
Some items are bespoke — individually customised with names, numbers, etc. These can have their own per-item Deco job ID separate from the main order job. Look for the MTO badge. You can assign individual job IDs in the expanded order view.

Auto-Match, Scan & Manual Link

Auto-Scan (Find Job)

Click Find Job on any order row. The system searches DecoNetwork by customer name, order details, and learned patterns. Results appear with a confidence score:

ScoreMeaning
80%+High confidence — very likely a correct match. Safe to confirm.
60–79%Medium confidence — probable match but verify the details before confirming.
<60%Low confidence — might not be correct. Check customer name, items, and dates carefully.

Bulk Scan

Select multiple orders → click SCAN in the toolbar. The system searches for matches for all selected orders at once. Confirm or reject each suggestion.

Manual Link

If you know the Deco job number, click the pencil icon or Enter Job ID and type the 6-digit number directly.

Learn Patterns

Every time you confirm a match, the system learns the pattern (SKU mappings, vendor rules, naming conventions) and uses it to improve future suggestions.

Item Mapping & Completion %

Once a Deco job is linked, the next step is item mapping — linking each Shopify line item to a specific product within the Deco job.

Why Map Items?

Without mapping, the system only knows the job exists but can't track which items are complete. Mapping enables:

  • Per-item production tracking (workflow ticks)
  • Accurate completion percentages
  • Stock readiness calculations
  • Identification of stuck items

The Completion Bars

In the Items Ready column, you'll see up to three progress bars:

BarColourWhat It Tracks
MAPPEDIndigoPercentage of items linked to Deco products. 100% = all items mapped.
STOCKBluePercentage of items with physical stock available in your warehouse.
MTOPurplePercentage of make-to-order items that have completed production.
⚠️ Mapping Gaps
If the MAPPING GAPS summary card shows a number, those orders have a Deco job linked but items aren't mapped. Open each one and use the mapping feature to connect the line items. Until mapped, the completion % will show 0%.

FULFILL Tab — Batch Ship Dashboard

Fulfill Tab

The FULFILL tab is your shipping command centre for getting orders out the door.

Ship-Ready Dashboard

By default, this shows only orders where production is 100% complete. Toggle to see all unfulfilled orders.

Batch Shipping Workflow

1
Select orders — Tick checkboxes for orders you're about to ship. Use "Select All" for everything visible.
2
Print packing slips — Click Print Packing Slips to generate printable sheets for all selected orders with product images and QC checklists.
3
Print & Ship All — Click this to create ShipStation shipping labels for all selected orders in one batch.
4
Choose carrier & service — Select your carrier (e.g., Royal Mail, DPD, Hermes) and service level from the dropdown.
5
Set weight — Enter the package weight (defaults to 250g). Toggle between grams and kilograms.
6
Confirm & create labels — The system creates labels in ShipStation, marks orders as fulfilled in Shopify, and sends tracking emails to customers.

Batch Results

After batch shipping, you'll see a result for each order:

  • Success — Label created, tracking number assigned
  • Failed — Error message explains why (missing address, invalid weight, etc.)

Address Validation

The system checks each order's shipping address before creating labels. If anything is missing (recipient name, street, city, postcode, country, phone), a yellow warning banner appears. The system will attempt to pull the address from ShipStation as a fallback.

ℹ️ Retry Failed Labels
If some labels fail (e.g., due to address issues), fix the address in Shopify or ShipStation, then use the Retry Failed button to re-attempt just the failed ones.

Print Order Sheets & Packing Slips

Order sheets are full-page printable production documents designed to travel with the order through your production floor.

What's On an Order Sheet

  • Order number, customer name, date
  • Club/team name and tags
  • Product images (loaded from Shopify)
  • Item name, SKU, size, colour, quantity
  • Personalisation details (names, numbers, custom text)
  • Production method (Embroidery, Print, DTF, etc.)
  • Deco Job ID and production status
  • Shipping address
  • QC checklist boxes for production sign-off

Three Ways to Print

MethodWhereHow
Single Order Expanded order sidebar Click the Print Order Sheet button.
Batch (Dashboard) Dashboard selection toolbar Select orders → click PRINT in the bottom toolbar.
Batch (Fulfill) FULFILL tab Select orders → click Print Packing Slips.
💡 Tip
Batch prints combine all selected orders into one print window with automatic page breaks between each order. The system waits for all product images to load before opening the print dialog.

Reports Tab Overview

Reports Tab

Scroll down the Reports tab to access all analytics. Each report auto-calculates from your live data.

ReportWhat It Shows

Bottleneck Analysis Identifies where orders are getting stuck in the production pipeline. Shows how long orders spend at each stage.
Completion Tracker Order completion rates against due dates. Tracks on-time vs late delivery performance.
Deco Notes Intelligence Analyses patterns in DecoNetwork order notes. Surfaces recurring issues and common instructions.
Payment Status Sync Payment status visibility across Shopify and DecoNetwork. Flags any payment discrepancies.
Capacity Forecast Forecasts production capacity based on current orders, historical velocity, and upcoming due dates.
Seasonal Demand Calendar Three views: Heatmap (GitHub-style 52-week grid), Monthly (average orders per month with year-over-year comparison), and Day of Week (busiest days bar chart with revenue). Click any cell for detail.

Stock Management

Stock Tab

The Stock tab manages your physical inventory — the blank garments and pre-printed items sitting in your warehouse.

Key Features

  • Physical Stock — Track quantities of items you hold. Add items manually or import via CSV.
  • Return Stock — Items returned by customers, tracked separately for quality processing.
  • Reference Products — Master product catalogue with SKUs, EANs, and vendor details.
  • Reorder Points — Set minimum stock levels. Get alerted when quantities drop below threshold.
  • EAN Barcodes — Barcode scanning support for stock intake and verification.

Stock levels feed into the Dashboard's STOCK % completion bar, showing which order items have physical stock ready for picking.

Notes, Chat & Notifications

Order Notes

Every order has a notes panel (click the 💬 icon or "Notes" in the expanded view). Notes are synced to the cloud via Firebase so all team members see the same information.

Features

  • @Mentions — Type @ to see a list of team members. Select someone to notify them about this order.
  • Priority Levels — Tag notes as Urgent, Info, or Action Needed.
  • Note Count Badge — Orders with notes show a count badge (e.g., "1 NOTE") in the table row.
  • Cloud sync — All notes sync across devices in real-time.

Notification Bell 🔔

The bell icon in the top navigation shows unread notifications. You receive notifications when:

  • Someone @mentions you in an order note
  • A reply is posted on an order you've commented on
  • Alert rules you've configured are triggered

Click the bell to see your notifications. Click any notification to jump directly to that order.

ℹ️ Sound Alert
New notifications play a subtle sound alert. This works on desktop and mobile (if the PWA is installed).

Syncing & Data Refresh

The app pulls data from three sources: Shopify, DecoNetwork, and ShipStation. Here's how each sync type works:

ButtonWhat It DoesWhen To Use
STATUS REFRESH Updates Shopify fulfilment status and Deco production status for active orders only. Fast — just checks for status changes. When you need the latest status but don't need new orders. Good for mid-day checks.
SYNC Full sync: Pulls 120 days of Shopify orders + 600 latest Deco jobs + ShipStation shipments. Catches new orders and recent changes. Start of day, or when new orders have been placed.
DEEP SCAN Extended sync: 365 days of Shopify orders + full Deco job history. Takes longer but catches everything. When you suspect missing orders, or once a week for a thorough refresh. Requires confirmation before running.
⚠️ Important
While syncing, you'll see a Syncing... indicator next to the tab buttons. The data may update in real-time as it comes in. Don't navigate away during a Deep Scan — it needs to complete fully.

Automatic Updates

The "Awaiting Deco Data..." and "Awaiting Deco Detail..." statuses indicate the app is background-loading detailed production information. These resolve automatically — no action needed.

Quick Reference — Status Cheat Sheet

I see this status...It means...I should...
Not Ordered No Deco job exists Create a job in DecoNetwork, then link it here
Awaiting Deco Data... App is syncing — temporary Wait. If stuck, hit STATUS REFRESH
