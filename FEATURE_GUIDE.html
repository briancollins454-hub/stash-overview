Dashboard & Core Navigation

The unified order table, filters, print sheets, notes, and exports

📊

Dashboard — Order Table & Filtering Dashboard

The Dashboard shows a unified table of every Shopify order merged with its matched DecoNetwork job. Each row shows order number, customer name, club/team tag, Shopify fulfillment status, DecoNetwork job ID, production workflow ticks (Ordered → Received → Produced → Shipped), item count, and total in £. Click any row to expand it and see full line-item detail with the sidebar.

Filtering & Searching
1
Type in the Search bar at the top — it searches by order number, customer name, SKU, and email. Input is debounced (300ms delay) so just type naturally.
2
Use the Filter Tags dropdown to only show orders tagged with specific clubs/teams (e.g. "AFC Southgate", "Barnet FC"). Tags come from Shopify order tags.
3
Click the coloured stat badges in the summary bar — these are quick filters: NOT ON DECO (unlinked), IN PRODUCTION, READY TO SHIP (100% complete), etc. Click the active badge again to clear the filter.
4
Use the Date Range picker to narrow by order creation date.
5
Click any column header (e.g. Date, Order #, Dispatch, Items Ready) to sort. A small arrow icon shows current sort direction. Click again to toggle asc/desc.
Expanding an Order Row
1
Click any order row — it expands to show all line items in a detail table: product name, SKU, size, quantity, unit price, personalisation properties, and per-item production status.
2
The right sidebar shows: customer email, Shopify order link (opens in new tab), DecoNetwork job ID with link, completion %, SLA target date, days remaining, and action buttons.
3
The workflow ticks in each row show 4 stages (Ordered ✓, Received ✓, Produced ✓, Shipped ✓). Green tick = done, orange = in progress, grey = not started.
💡 URL routing is built in — every tab updates the URL (e.g. ?tab=dashboard). Bookmark any tab or use browser back/forward to navigate.

Visual board showing all unfulfilled orders in status columns. Only orders where fulfillment status is not "fulfilled" appear. Each card shows order number, customer name, club tag, total items, and a completion percentage bar.

Click KANBAN in the top navigation bar.

How to Use

Print Order Sheet Dashboard

Generates a detailed, print-optimised production sheet for any single order. Opens in a new browser window and triggers the print dialog automatically. The sheet includes product images (thumbnails pulled from Shopify), personalisation properties, production method detection, pick/pack checkboxes, QC checklist, internal notes, and rush order highlighting.

How to Use
1
On the Dashboard, click an order row to expand it.
2
In the right-hand sidebar, click the "Print Order Sheet" button (has a 🖨 Printer icon).
3
A new browser window opens with the formatted sheet. The print dialog appears automatically.
4
Print it physically, or use "Save as PDF" in the print dialog to save a copy.
What's on the Sheet
•
Rush Banner: If the order has a rush, urgent, priority, or express tag, a bold red "🚨 RUSH ORDER" banner appears at the top.
•
SLA Countdown: Large box showing days remaining. Red with "OVERDUE" text and red border if past SLA, green with "X DAYS REMAINING" and green border if on track.
•
Order Details Grid (3 columns): Fulfillment status, production status, DecoNetwork job ID, total (£), item count, customer email, estimated production method, produced/total count, and completion %.
•
Stock Items Table: Products not flagged as MTO. Columns: IMG (40×40px thumbnail from Shopify), Item Name (with personalisation properties shown as grey badges underneath — e.g. "Name: SMITH", "Number: 7"), Qty, SKU, Method (auto-detected — see below), Status, and a Pick ☐ checkbox column for the production floor to tick off.
•
MTO Items Table: Same layout but only for made-to-order items (items with "mto" in the name). Shown separately so the team knows which items require production before dispatch.
•
QC Checklist: 6 checkboxes in a 3×2 grid for the floor to tick: Print Aligned, Correct Size, Colour Match, Packaging, Labels Attached, Final Sign-off.
•
Internal Notes: If any internal notes have been saved for this order (via the Notes feature), they appear on a yellow background with author email and timestamp per note.
•
Production Notes: A blank dashed-border area for handwritten notes on the printed sheet.
Auto-Detected Production Methods
•
The system scans each item's product name and SKU for keywords and assigns a method:
"embroid" → Embroidery  |  "screen" / "silk" → Screen Print  |  "dtf" → DTF
"dtg" / "direct to garment" → DTG  |  "vinyl" / "heat press" / "transfer" → Heat Press
"sublim" → Sublimation  |  "print" / "personalise" / "custom" → Print (General)
If no keywords match, it shows "—"
💡 The sheet header shows "STASH SHOP" branding plus the exact date/time it was printed. The sheet is formatted for A4 and includes page-break-friendly CSS so multi-page orders print cleanly.

Section 02

Intelligence

Auto-matching, duplicate detection, and production forecasting — all on the INTEL tab

🤖

Auto-Match Engine Intel

Fuzzy-matches Shopify line items to DecoNetwork job items using product name, SKU, size, and colour similarity. Shows a confidence percentage per match. Learns from your confirmed matches via a product mappings dictionary stored in localStorage — the more you confirm, the better it gets.

How to Use
1
Click INTEL in the top nav. Auto-Match runs automatically on page load.
2
Each unmatched order shows suggested matches with a confidence % (e.g. 87% match).
3
Click "Apply" on a suggested match to confirm it. This links the Shopify item to the DecoNetwork job item and saves the product mapping for future auto-matching.
4
Click "Dismiss" to reject a bad suggestion. The engine will try different matches next time.
5
Click any order number link to jump to that order on the Dashboard.
💡 The matching engine checks product mappings first (instant), then falls back to fuzzy name/SKU comparison. Confirmed matches become instant in future syncs.
🔍

Duplicate Detection Intel

Scans for potential duplicate orders by comparing customer names, order totals, and creation dates within a close timeframe. Flags suspicious pairs to prevent double-processing.

How to Use
1
Navigate to INTEL — the duplicate scanner runs automatically alongside Auto-Match.
2
Flagged pairs show why they were flagged — e.g. "Same customer, £45.00 vs £45.00, 1 day apart".
3
Click an order number to inspect it on the Dashboard.
4
Click "Dismiss" to mark as not-duplicate. Dismissed pairs won't be flagged again.
📈

Production Forecasting Intel

Analyses historical order patterns to forecast upcoming weekly workload. Groups unfulfilled orders by their SLA target / due date to predict how many orders and units need producing each week.

How to Use
1
Navigate to INTEL — the forecast panel loads automatically.
2
View the weekly breakdown showing predicted order volumes and unit counts.
3
Use this to plan staffing and material ordering for upcoming peak weeks.

Production Tab

Priority management, calendar planning, and batch printing — all under the PRODUCTION tab

🔥

Priority Queue Production

A dedicated priority view combining two sources: (1) orders you manually flag as urgent/high priority, and (2) orders auto-detected from Shopify tags (rush, urgent, priority, express). Only unfulfilled orders appear. Sorted by priority level (urgent first), then by due date.

Viewing the Queue
1
Click PRODUCTION in the top nav. The Priority Queue appears at the top of the page with a red 🔥 icon and a badge showing count.
2
Each row shows: priority badge (🔥 URGENT red or ⚠ HIGH amber), order number (clickable), customer name, club tag (indigo badge), completion %, due date (with clock icon), and optional note in italics.
3
Rows have a coloured left border — red for urgent, amber for high.
4
Auto-detected tag-based orders show a small grey "TAG" label so you know they came from Shopify tags, not manual flagging.
5
Click any order number to jump to that order on the Dashboard.
6
If the queue is empty, a green Zap icon shows with "No priority orders" and a hint to flag orders or add Shopify tags.
Manually Flagging an Order
1
Click the red "Flag Order" button (top right of the Priority Queue panel). A form slides open with a light red background.
2
Fill in four fields:
Order #Type the order number (e.g. 1234 — without #). Must match an existing order in the data.
LevelDropdown: 🔥 Urgent or ⚠️ High
Due DateDate picker — set an explicit deadline for this order (optional)
NoteFree text — e.g. "Customer CEO called", "Match day Saturday" (optional)
3
Click "Add". The order enters the queue immediately, sorted by priority then due date.
4
To remove a manually flagged order, click the × button on its row. (Tag-based orders can't be removed here — remove the tag in Shopify instead.)
5
Click the × next to the form to cancel without adding.
💡 Priority flags persist in localStorage and record who flagged (setBy = your email) and when (setAt = timestamp). They survive page refreshes and browser restarts.

Production Calendar Production

A visual calendar plotting unfulfilled orders by their due date (uses productionDueDate or slaTargetDate). Each day shows an order count badge colour-coded by workload density. Supports month and week views with a heat-map colour scale.

Navigation Controls
1
On the PRODUCTION tab, scroll below the Priority Queue to find the Production Calendar (purple 📅 icon).
2
Use the ◄ ► arrow buttons to navigate between months (or weeks in week view). The current period shows as "March 2026" or "Week of 30 Mar".
3
Click "Week View" button to switch to a 7-day view (Monday–Sunday). The button text changes to "Month View" to switch back. In week view, individual order numbers are listed directly on each day cell.
4
Click "Today" (indigo text, centre) to snap back to the current date.
Interacting with Days
1
Click any day cell to select it — a detail panel appears below the calendar. Click the same day again to deselect.
2
The detail panel shows the date (e.g. "Monday, 31 March"), total orders, total units, then a scrollable list of orders with: order number, customer name, club, items, and completion %. Click an order to navigate to Dashboard.
3
Today's date always has an indigo ring border so it's easy to spot. Selected day gets a light indigo background.
Heat Map Colour Scale
•
Cell background: White = 0 orders  |  Lightest indigo = 1–3  |  Light = 4–8  |  Medium = 9–15  |  Dark indigo = 16+
•
Count badge colours: Indigo = 1–5 orders  |  Amber = 6–10  |  Red = 11+
•
A legend bar is shown below the calendar grid: Less □□□□□ More
💡 Greyed-out dates outside the current month are still clickable. Use the heat map to spot bottleneck days and redistribute workload before they happen.

Batch Print Sheets Production

Groups all unfulfilled line items (with remaining quantity > 0) across all orders by production method, club, or due date — then prints grouped production sheets for the floor. quantity - fulfilledQuantity is calculated so already-shipped items don't appear. Rush orders are sorted to the top and marked with ⚡ on the printed output.

Grouping & Filtering
1
On the PRODUCTION tab, scroll below the Calendar. The header shows the indigo 🖨 Printer icon plus "N lines, N units" count of remaining work.
2
Use the "By Method" dropdown (top-right) to choose grouping mode:
• By Method — Embroidery, DTF, Screen Print, DTG, Heat Press, Sublimation, Print (General), Stock / No Decoration
• By Club — groups by club/team tag (or "No Club")
• By Due Date — groups by SLA target date (or "No Date")
3
Below the dropdown is a method filter bar with checkbox-toggle buttons for each detected method. Each button has a ☑/☐ icon. Click to toggle methods on/off — only checked methods appear in the list. The filter bar shows a 🔽 Filter icon on the left.
Viewing & Expanding Groups
1
Each group shows as a collapsible row: ▶ EMBROIDERY — 12 lines with unit count on the right. Click to expand/collapse.
2
Use "Expand All" and "Collapse" buttons (top-right) to toggle all groups at once.
3
Inside each expanded group, a table shows: Order (number), Customer, Item (product name), SKU, Size (auto-extracted from item properties if key contains "size"), Qty (remaining), Job ID (DecoNetwork), and Due (date). Rush items show with a ⚡ icon.
Printing
1
Click the indigo "🖨 Print" button (top-right). A new browser window opens.
2
The printed output has one section per group, each with:
• Group heading showing method/club/date, line count, and unit total
• Table with columns: Order, Customer, Item, SKU, Size, Qty, Job ID, Due, Done ✓
• Rush items highlighted with yellow background and ⚡ suffix
• Per-group unit total at the bottom
3
Footer shows "Generated [datetime] — Stash Shop Sync". Print dialog opens automatically.
💡 The "Done ✓" column is an empty cell on the printed sheet — meant for the floor team to tick by hand. Size is auto-extracted from personalisation properties where the property name contains "size".

Profitability Report Reports

Revenue breakdown per order with summary stats. Revenue comes from Shopify order total (£). Cost/margin columns are included but currently show "—" because DecoNetwork doesn't expose job cost data via its API — the columns are pre-built and ready for when cost data becomes available.

Summary Cards
1
Click REPORTS in the top nav. The Profitability Report appears first with 4 summary cards at the top:
Total RevenueSum of all order totals, shown in £ with 2 decimal places
OrdersTotal order count
Avg Order ValueTotal revenue ÷ order count
Cost Data AvailableShows "0/N" with an amber ⚠ warning: "DecoNetwork doesn't expose cost data via API"
Order Table
1
Below the summary cards, a scrollable table shows every order with columns: Order #, Customer, Club (indigo text), Items (total quantity), Revenue (£), Cost ("—"), Margin ("—" with ↑↓ trend icons when data available), % (margin percentage).
2
Use the Sort By dropdown to change ordering: Sort by Margin, Sort by Revenue (highest first), or Sort by Order # (newest first).
3

4

Club / Team Leaderboard Reports

Ranks all clubs/teams by performance. Groups orders by club tag and shows per-club stats. Top 3 get medal icons: 🥇 (gold), 🥈 (silver), 🥉 (bronze). Remaining clubs show their rank number. Click to expand and see individual orders.

How to Use
1
On the REPORTS tab, scroll below Profitability. The header shows a 🏆 Trophy icon and "(N clubs)" count.
2
Use the Sort By dropdown to rank by:
• By Revenue — highest total spend at top
• By Order Count — most orders first
• By Item Count — most units first
• By Turnaround (fastest) — lowest average fulfillment days first (clubs with no fulfilled orders sort to bottom, show "—")

Click a club row to expand — a nested table shows that club's orders (up to 20, newest first) with: Order #, Customer, Date, Revenue (£), and Status badge (green "fulfilled" or amber "unfulfilled"). Each row is clickable to navigate to Dashboard.
5
If a club has more than 20 orders, a "+N more orders" note appears at the bottom of the expanded table.
💡 "No Club" appears for orders without a club/team Shopify tag. Turnaround average only includes orders that have actually been fulfilled — open orders don't affect it.

Late Order Report Reports

Automatically identifies at-risk orders based on three criteria. Only unfulfilled orders are checked. Each flagged order gets a severity level: 🔴 Critical or 🟡 Warning. When an order matches multiple criteria, the worst severity wins (deduplicated per order).

Issue Type What Triggers It 🟡 Warning 🔴 Critical
Unlinked No DecoNetwork job ID and order is 3+ days old 3–6 days old 7+ days old
Past SLA Today's date is after the order's SLA target date 1–4 days overdue 5+ days overdue
Stuck in Prod Has a DecoNetwork job, less than 100% complete, and 10+ days in production 10–14 days 15+ days
How to Use
1
On the REPORTS tab, scroll to Late Order Report. The ⚠ header shows a red badge with total count of flagged orders.
2
Summary badges below header: 🔴 N Critical (red) and 🟡 N Warning (amber).
3
Use the filter dropdown to narrow by issue type. Each option shows its count:
• All Issues (N) | Unlinked 3+ Days (N) | Past SLA (N) | Stuck Production (N)
4
Table columns: Severity icon (🔴/🟡), Order #, Customer, Club, Issue (description text like "Unlinked for 5 days" or "3 day(s) past SLA target"), Days count (in red), Completion % bar (visual progress), Job ID. Critical rows have a light red background tint.
5
Click "Export CSV" button to download the filtered list as late-orders-YYYY-MM-DD.csv with columns: Order, Customer, Club, Days Late, Reason, Severity, Completion %.
6
Click any row to navigate to that order. 25 rows shown by default — click "Show All" to see everything.
💡 If there are zero late orders, a green "🎉 No late orders — everything on track!" message appears with a clock icon. Address 🔴 critical issues first — they indicate orders that are actively late or stuck.

🔄

Returns & Remake Tracker Ops

Full return/remake lifecycle tracking with reason codes, 5-step status workflow, and remake job linking. Persists in localStorage. Summary stats track defect patterns across returns.

Logging a Return
1
Click OPS in the top nav. The Returns & Remakes panel is first (🔄 icon). The header shows an orange badge: "N open" (returns not yet resolved).
2
Click the orange "Log Return" button (has a ➕ icon).
3
A form slides open on a light orange background with a 4-column grid:
Order #Type the order number. The system auto-looks up the order — if found, the customer name appears in green below the field. If not found, "Order not found" shows in red.
ItemIf the order was found, this becomes a dropdown populated with all line items from that order (by product name). Select the returned item. If no match, stays as a free-text field.
SizeFree text — enter the size of the returned item
QtyNumber input — defaults to 1
ReasonDropdown with 6 options:
Wrong Size  |  Print Defect  |  Wrong Item Sent  |  Damaged in Transit  |  Customer Changed Mind  |  Other
DetailFree text — additional notes (optional), spans 2 columns
4
Click "Save". The return is created with status "Received" and appears in the list. The "Save" button is disabled until both an order match and an item are selected.
Managing Returns
1
Each return row shows: order # (clickable), customer name, status badge (coloured — red/amber/blue/purple/green), reason badge (coloured text), item name, size, quantity, detail note in italics, and "Logged [date] by [email]" at bottom.
2
Use the status dropdown on each row to progress through the 5-step workflow:
ReceivedItem received back — initial state
InspectedItem has been inspected/assessed
Remake OrderedReplacement has been ordered through DecoNetwork
Remake ShippedReplacement has been shipped to customer
ResolvedReturn is complete — status dropdown disappears
3
Click the ✏️ pencil icon on any row to enter a Remake Job ID — a small inline form appears with a text input and ✓ confirm button. The linked job ID then shows in blue: "Remake Job: #12345".
4
Filter returns using the top dropdown: All (N), Received (N), Inspected (N), Remake Ordered (N), Remake Shipped (N), Resolved (N).
5
Delete a return with the 🗑 trash icon.
Summary Stats Bar
•
Below the header: Total: N (all-time returns)  |  Defects: N (reason = Print Defect)  |  Wrong Size: N. Track these to spot recurring quality issues.

Section 06

Settings & Configuration

API credentials, auto-refresh, holidays, and cloud sync

⚙️

Settings Panel

Access by clicking the Gear icon (⚙) in the top nav bar. The settings panel slides open with all configuration options. All changes save to localStorage immediately.

Configuration Sections
1
Shopify API: Enter your Shopify store domain (e.g. yourstore.myshopify.com) and Admin API access token. Used for all GraphQL queries to fetch orders, line items, images, and custom attributes.
2
DecoNetwork API: Enter your DecoNetwork subdomain URL, API key, and credentials. Used for fetching job data, item statuses, and matching.
3
Auto-Refresh Interval: Set the sync interval in minutes (default: 5). Orders re-fetch from both APIs at this interval. Also re-syncs when the browser tab regains focus after 2+ minutes away.
4
SLA Target Days: Number of business days for SLA calculations, used to compute each order's slaTargetDate.
5
Holiday Ranges: Add date ranges for business closures (e.g. Christmas week). SLA calculations skip these non-working days.
6
Supabase Cloud Sync: Enter Supabase project URL and anon key to sync notes, returns, artwork approvals, and shipping labels across devices.
7
Google Gemini API Key: Required for the ANALYST tab's AI process analysis feature.
